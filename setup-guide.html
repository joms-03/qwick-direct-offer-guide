How We Built This Website Guide

This is the direct, professional version of the setup guide. It explains the full process from a fresh computer to a published website, including the apps, tools, and steps we used.

Plain HTML site No complex framework required Free GitHub Pages hosting

What this guide is for

This guide is for someone who is starting from zero. That includes someone who just downloaded Claude and does not yet have Visual Studio Code. The goal is simple: get from a new machine to a live website with clear instructions and no unnecessary complexity.

If you already have Claude, that is helpful. But you still need Visual Studio Code to create, edit, and save the web files.

What you need

Required

  • Visual Studio Code
  • A GitHub account
  • A browser such as Chrome, Edge, or Safari
  • A folder on your computer for the project

Helpful but optional

  • Claude for writing and refining content
  • GitHub Desktop if you want a simpler interface
  • Live Server extension in VS Code for easy preview

Step-by-step process

1Install the tools

Start with the essentials.

  • Download and install Visual Studio Code.
  • Create a GitHub account if you do not already have one.
  • If you use Claude, install it as well. It is optional, but useful for drafting content.
For this project, Visual Studio Code is the most important tool. Without it, you cannot properly edit the website files.

2Open a project folder in Visual Studio Code

Create a folder on your computer for the website. Then open that folder in VS Code.

  • Choose a simple name such as my-website.
  • In VS Code, go to File → Open Folder.
  • Select the project folder you created.

3Create the main file

The website starts with one HTML file.

  • Create a file named index.html.
  • Place it directly inside the project folder.
  • Start with the basic structure: <!DOCTYPE html>, <html>, <head>, and <body>.

4Build the page content

For this type of site, the content is written directly into the HTML file. You can structure it with headings, sections, cards, buttons, and simple styling.

  • Add a title and short description.
  • Create sections for the overview, steps, and important notes.
  • Add simple CSS so the page looks clean and professional.

5Preview the site locally

You should check the page in a browser before publishing it.

  • Open the HTML file in your browser.
  • Confirm the layout, text, spacing, and styling look correct.
  • Make changes in VS Code and refresh the browser as needed.
If you want a smoother preview experience, install the Live Server extension in VS Code.

6Upload the website to GitHub

Once the page looks correct, put it into GitHub so it can be published.

  • Open the project folder in VS Code.
  • Open the Source Control panel.
  • Stage the file, add a commit message, and push it to GitHub.

7Publish with GitHub Pages

This is the final step that makes the website live on the internet.

  • Go to your repository on GitHub.
  • Open Settings → Pages.
  • Choose the main branch as the source.
  • Save the settings and wait a few minutes for the site to publish.

8Share the link

Once GitHub Pages finishes, you will have a public URL for the website.

  • Use the GitHub Pages URL in Slack, email, or a document.
  • If the page does not update right away, wait a few minutes and refresh the page.

What makes this approach simple

Why this works

  • It uses plain HTML, which is easy to learn and maintain.
  • GitHub Pages provides free hosting with minimal setup.
  • Visual Studio Code is enough to build and edit the site.

What to remember

  • Claude can help write content, but VS Code is still required.
  • GitHub is where the site is stored and published.
  • Keep the project folder organized and name files clearly.
This guide is designed to be practical, direct, and easy to follow from a fresh setup.
